## 다이어그램 1: 3-Layer 시스템 구조 (메인)

### 레이아웃

```
┌─────────────────────────────────────────┐
│  User Space (Host) - 연한 파랑색        │
│  ┌───────────────────────────────────┐  │
│  │ Mini-KVM VMM (1,500 LOC)          │  │
│  │                                   │  │
│  │ • VM/vCPU 관리                     │  │
│  │ • I/O 에뮬레이션 (UART, Hypercall) │  │
│  │ • Interrupt 주입                  │  │
│  │ • pthread 멀티스레딩               │  │
│  └───────────────────────────────────┘  │
│                                         │
│            ⬇ KVM ioctl()                 │
├─────────────────────────────────────────┤
│  Kernel Space - 연한 초록색              │
│  ┌───────────────────────────────────┐  │
│  │ Linux KVM Module                  │  │
│  │                                   │  │
│  │ • 하드웨어 가상화 (Intel VT-x)      │  │
│  │ • EPT 메모리 관리                  │  │
│  │ • VM Exit 처리                    │  │
│  └───────────────────────────────────┘  │
│                                         │
│            ⬇ Hardware                   │
├─────────────────────────────────────────┤
│  Guest VMs - 연한 핑크색                 │
│  ┌───────────────────────────────────┐  │
│  │ Real Mode (16-bit) 게스트          │  │
│  │ OR                                │  │
│  │ Protected Mode (32-bit) + 1K OS   │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

### PowerPoint 제작 단계

1. **도형 삽입**:
   - 사각형 (모서리 둥글게) 3개
   - 각 레이어마다 하나씩

2. **색상 설정**:
   - User Space: 연한 파랑 (#E3F2FD)
   - Kernel Space: 연한 초록 (#E8F5E9)
   - Guest VMs: 연한 핑크 (#FCE4EC)
   - 테두리: 진한 회색 (#424242)

3. **텍스트 배치**:
   - 각 상자 내부에 제목과 bullet points
   - 글꼴: 맑은 고딕 또는 Arial, 크기 10-12pt
   - 제목은 Bold

4. **화살표**:
   - "KVM ioctl()" 화살표: 굵은 화살표, 양방향
   - "Hardware" 화살표: 굵은 화살표, 아래 방향
   - 색상: 검은색 또는 진한 회색

5. **강조**:
   - "1,500 LOC" 부분은 다른 색으로 강조 (빨간색 또는 핑크)
   - Intel VT-x, EPT 등은 Bold

## 다이어그램 2: Multi-vCPU 구조 (선택적)

### 레이아웃

```
┌───────────────────────────────────────────┐
│          Mini-KVM VMM (Host)              │
│                                           │
│  ┌─────────┬─────────┬─────────┬─────────┐│
│  │ vCPU 0  │ vCPU 1  │ vCPU 2  │ vCPU 3  ││
│  │ Thread  │ Thread  │ Thread  │ Thread  ││
│  │         │         │         │         ││
│  │ Cyan    │ Green   │ Yellow  │ Blue    ││
│  └─────────┴─────────┴─────────┴─────────┘│
│             ⬇  ⬇  ⬇  ⬇                    │
│  ┌───────────────────────────────────┐    │
│  │   Shared VM (KVM)                 │    │
│  │                                   │    │
│  │   Memory Isolation:               │    │
│  │   0x00000-0x3FFFF (vCPU 0)        │    │
│  │   0x40000-0x7FFFF (vCPU 1)        │    │
│  │   0x80000-0xBFFFF (vCPU 2)        │    │
│  │   0xC0000-0xFFFFF (vCPU 3)        │    │
│  └───────────────────────────────────┘    │
└───────────────────────────────────────────┘
```

### PowerPoint 제작 단계

1. **vCPU 박스**:
   - 4개의 작은 사각형을 가로로 배치
   - 각각 Cyan, Green, Yellow, Blue 배경색
   - 텍스트: "vCPU 0-3", "Thread", 색상 이름

2. **VM 박스**:
   - 큰 사각형 하나
   - 연한 회색 배경 (#F5F5F5)
   - 메모리 주소 범위 표시

3. **화살표**:
   - 4개 vCPU에서 VM으로 내려오는 화살표
   - 각각 해당 색상으로

## 다이어그램 3: Hypercall 플로우 (선택적)

### 레이아웃

```
┌─────────┐         ┌─────────┐         ┌─────────┐
│ Guest   │  out    │  KVM    │ ioctl   │  VMM    │
│ Program │ ──────> │ Module  │ ──────> │ Handler │
│         │ port    │         │ return  │         │
│         │ <────── │         │ <────── │         │
└─────────┘         └─────────┘         └─────────┘

Hypercalls:
• HC_PUTCHAR (0x01): 문자 출력 (BL)
• HC_GETCHAR (0x02): 문자 입력 (AL)
• HC_EXIT (0x00): 게스트 종료
```

### PowerPoint 제작 단계

1. **박스 3개**:
   - Guest Program (핑크)
   - KVM Module (초록)
   - VMM Handler (파랑)

2. **화살표**:
   - Guest → KVM: "out port 0x500"
   - KVM → VMM: "ioctl (VM_EXIT_IO)"
   - VMM → KVM: "return"
   - KVM → Guest: "resume"

3. **Hypercall 목록**:
   - 하단에 bullet points로 표시
   - 번호와 기능 명시

## 다이어그램 4: 듀얼 모드 비교 (선택적)

### 테이블 형식

| 항목 | Real Mode | Protected Mode |
|------|-----------|----------------|
| **비트 수** | 16-bit | 32-bit |
| **주소 공간** | 1 MB | 4 GB |
| **페이징** | 없음 | 4MB PSE |
| **세그먼트** | CS:IP | GDT/IDT |
| **인터럽트** | 없음 (IRQCHIP OFF) | IRQ0/IRQ1 지원 |
| **게스트** | 6개 프로그램 | 1K OS (9개 프로그램) |

### PowerPoint 제작 단계

1. **표 삽입**:
   - 6행 3열 표
   - 헤더 행 배경색: SOFTCON 핑크
   - 데이터 행 배경색: 흰색, 교차로 연한 회색

2. **강조**:
   - Real Mode IRQCHIP OFF - 빨간색 또는 Bold
   - Protected Mode의 GDT/IDT, 페이징 - 파란색 또는 Bold

## 색상 팔레트 (일관성 유지)

### 기본 색상
- **SOFTCON 핑크**: #E91E63 (제목, 강조)
- **User Space**: #E3F2FD (연한 파랑)
- **Kernel Space**: #E8F5E9 (연한 초록)
- **Guest VMs**: #FCE4EC (연한 핑크)

### vCPU 색상
- **vCPU 0**: Cyan (#00BCD4)
- **vCPU 1**: Green (#4CAF50)
- **vCPU 2**: Yellow (#FFEB3B)
- **vCPU 3**: Blue (#2196F3)

### 보조 색상
- **테두리**: #424242 (진한 회색)
- **배경**: #FFFFFF (흰색)
- **텍스트**: #000000 (검은색)

## PowerPoint 도형 팁

### 사각형 만들기
1. 삽입 → 도형 → 사각형
2. 모서리 둥글게: 도형 서식 → 효과 → 부드러운 가장자리
3. 그림자 추가: 도형 서식 → 효과 → 그림자 (미세하게)

### 화살표 만들기
1. 삽입 → 도형 → 블록 화살표 또는 선 화살표
2. 두께: 2-3pt
3. 스타일: 단색, 끝에 화살표

### 텍스트 정렬
1. 도형 내부 클릭 → 텍스트 입력
2. 중앙 정렬 (Ctrl+E) 또는 왼쪽 정렬
3. 여백: 도형 서식 → 텍스트 상자 → 여백 조정

## 배치 가이드

### 3-Layer 다이어그램 (메인)
- **위치**: "주요기술 & 아키텍처" 섹션
- **크기**: 중간~큰 사이즈 (포스터의 15-20%)
- **위치**: 섹션 중앙 또는 우측

- **위치**: "개발내용" 섹션
- **크기**: 작음~중간
- **Multi-vCPU 스크린샷과 함께 배치**
