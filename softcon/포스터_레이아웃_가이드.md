### ① 개발동기 및 목적 (좌측 상단, 20%)

**제목**: "왜 Mini-KVM을 개발했는가?"
- 헤더 배경: 진한 핑크 (#E91E63)
- 헤더 텍스트: 흰색, 18-20pt, Bold

**내용**:
```
문제 인식
• QEMU는 100만 줄 이상의 복잡한 코드
• 가상화 기술 학습 진입 장벽이 매우 높음
• 교육용 VMM은 기능 제한적

해결 방안
• 1,500줄의 간결한 C 코드
• Linux KVM으로 네이티브급 성능
• Real + Protected Mode 지원

핵심 목표
✓ 가상화 핵심 개념을 실제로 구현하며 학습
✓ 작지만 완전한 시스템으로 교육 가치 극대화
```

**강조 수치**:
- "1,500줄" - 빨간색 또는 박스
- "네이티브급 성능" - Bold

---

### ② 주요기술 & 시스템 아키텍처 (우측 상단, 25%)

**제목**: "기술 아키텍처"

**메인 비주얼**: 3-Layer 시스템 다이어그램
- 크기: 섹션의 60%
- 위치: 중앙

**핵심 기술 (4가지)**:
```
1. 듀얼 모드 지원
   Real (16-bit) + Protected (32-bit)

2. Multi-vCPU 병렬 실행
   최대 4개 vCPU, pthread 기반

3. 조건부 IRQCHIP
   Real Mode 10배 성능 향상

4. 하이퍼콜 인터페이스
   PUTCHAR, GETCHAR, EXIT
```

**기술 스택** (아이콘 또는 텍스트):
- C Language
- Linux KVM
- pthread
- GNU Binutils
