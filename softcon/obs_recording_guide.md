# OBS Studio 녹화 가이드 - Mini-KVM 소프트콘 영상

## OBS Studio 설치 및 초기 설정

### 설치 (이미 설치되어 있다면 스킵)
```bash
sudo apt install obs-studio
```

### 첫 실행 설정
1. OBS Studio 실행
2. 자동 구성 마법사 → "영상 녹화에 최적화" 선택
3. 해상도: 1920x1080 (1080p)
4. FPS: 30 또는 60 (60 권장)

---

## 씬(Scene) 구성

### 씬 1: 타이틀 카드
**용도**: 인트로 (0:00-0:05)

1. 새 씬 생성: "1_Title"
2. 소스 추가 → "이미지"
   - 이름: "Thumbnail"
   - 파일: `202322071_설규원_소프트콘_썸네일.png`
   - 변환: 화면에 맞게 확대

---

### 씬 2: 슬라이드 (Marp)
**용도**: 문제 정의, 아키텍처, 성능 비교, 응용 분야, 클로징 슬라이드

1. 새 씬 생성: "2_Slides"
2. **방법 A**: 이미지 슬라이드 사용
   - Marp에서 이미지로 export: `npx @marp-team/marp-cli presentation.md -o slides/slide-%d.png`
   - 각 슬라이드를 개별 이미지로 추가
3. **방법 B**: 브라우저 소스 사용 (추천)
   - Marp HTML 생성: `npx @marp-team/marp-cli presentation.md -o presentation.html`
   - 소스 추가 → "브라우저"
   - URL: `file:///home/seolcu/문서/코드/mini-kvm/softcon/presentation.html`
   - 너비: 1920, 높이: 1080
   - 단축키로 슬라이드 전환 (→ 키)

---

### 씬 3: 터미널 풀스크린
**용도**: Real Mode 데모, Protected Mode 데모 (1:00-3:30)

1. 새 씬 생성: "3_Terminal_Fullscreen"
2. 소스 추가 → "화면 캡처" 또는 "창 캡처"
   - 이름: "Terminal"
   - 창: 터미널 창 선택
   - 변환: 화면에 맞게 크롭/확대
3. **터미널 설정** (녹화 전 준비)
   ```bash
   # 터미널 프로파일 설정
   # 폰트 크기: 24pt 이상
   # 배경: 불투명 검정 (#000000)
   # 색상: xterm-256color
   ```
4. **텍스트 오버레이 추가** (선택적)
   - 소스 추가 → "텍스트 (FreeType 2)"
   - 위치: 하단 또는 상단
   - 폰트: Noto Sans KR Bold, 크기 36
   - 내용: "초기화: <5ms | 메모리: 1.5MB" 등

## 전환 효과 (Transitions) 설정

### 씬 전환 설정
1. 설정 → 전환
2. 기본 전환: "Fade" (페이드)
3. 전환 시간: 500ms

### 개별 씬 전환 사용자 정의
- 타이틀 → 슬라이드: Fade (1초)
- 슬라이드 → 터미널: Cut (즉시) - 역동성
- 터미널 → 차트: Fade (500ms)
- 차트 → 슬라이드: Fade (500ms)
- 슬라이드 → 클로징: Fade (1초)

## 오디오 설정

### 마이크 입력
1. 설정 → 오디오
2. 마이크/보조 오디오 → 마이크 선택
3. **필터 추가** (음질 개선)
   - 노이즈 억제: Speex 또는 RNNoise
   - 노이즈 게이트: -30dB 임계값
   - 컴프레서: 비율 3:1
   - 게인: +5dB (필요시)

### 배경 음악 (후반 작업에서 추가 권장)
- OBS에서 실시간 추가보다는 영상 편집 소프트웨어에서 추가 권장
- 저작권 없는 음악 소스:
  - YouTube Audio Library
  - Incompetech
  - Free Music Archive

## 녹화 설정

### 녹화 품질
1. 설정 → 출력
2. 출력 모드: "고급"
3. 녹화 탭:
   - 형식: mp4 (mkv도 가능, 안전)
   - 인코더: x264 (CPU) 또는 NVENC (NVIDIA GPU)
   - 비트레이트: 8000-12000 Kbps
   - 키프레임 간격: 2초
   - 사전 설정: "고품질" 또는 "무손실"

### 비디오 설정
1. 설정 → 비디오
2. 기본 해상도: 1920x1080
3. 출력 해상도: 1920x1080
4. FPS: 60 (터미널 애니메이션 부드럽게) 또는 30

## 문제 해결

### 터미널 캡처가 검은 화면으로 나옴
- **해결**: 창 캡처 대신 "화면 캡처" 사용
- 또는 터미널 컴포지터 비활성화

### 음성이 너무 작음
- **해결**: 마이크 입력 레벨 조정 (오디오 믹서에서 슬라이더)
- 필터 추가 → 게인 (+5~+10dB)

### 영상이 끊김 (프레임 드롭)
- **해결**: 인코더 사전 설정을 "빠르게" 또는 "초고속"으로 변경
- CPU 부하 줄이기: 다른 프로그램 종료

### 파일 크기가 너무 큼
- **해결**: 비트레이트 낮추기 (8000 → 5000 Kbps)
- 또는 후반 작업에서 재인코딩

### 슬라이드가 흐릿함
- **해결**: 브라우저 소스 해상도 확인 (1920x1080)
- Marp CSS 폰트 크기 조정

## 후반 작업 도구 추천

### Linux 영상 편집 소프트웨어

#### 1. **Kdenlive** (추천 - 오픈소스, 강력함)
```bash
sudo apt install kdenlive
```
**기능**:
- 멀티 트랙 편집
- 전환 효과 및 필터
- 텍스트 오버레이
- 오디오 믹싱
- 4K 지원

**사용법**:
- 프로젝트 생성 (1920x1080, 30 or 60fps)
- 세그먼트 클립 import
- 타임라인에 배치 및 트리밍
- 전환 효과 추가
- 텍스트/타이틀 추가
- 오디오 레벨 조정
- 렌더링 (H.264, 고품질)

#### 2. **DaVinci Resolve** (프로페셔널)
- 무료 버전 available
- 다운로드: https://www.blackmagicdesign.com/products/davinciresolve
- 강력한 색보정 및 오디오 믹싱

#### 3. **Shotcut** (간단함)
```bash
sudo snap install shotcut --classic
```

## 최종 렌더링 설정

### Kdenlive 렌더링
1. 프로젝트 → 렌더링
2. 프로파일: MP4 (H.264)
3. 품질: 고품질 (8000-12000 Kbps)
4. 오디오: AAC, 192 Kbps
5. 해상도: 1920x1080
6. FPS: 30 or 60
7. 렌더링 시작

### 예상 렌더링 시간
- 5분 영상: 10-30분 (컴퓨터 성능에 따라)

### 최종 파일 확인
- 크기: 50-150MB (예상)
- 해상도: 1920x1080
- 길이: 5분 (300초)
- 오디오: 동기화 확인
- 화질: 선명도 확인

### 파일 크기 최적화 (필요시)
```bash
# ffmpeg로 재인코딩 (크기 줄이기)
ffmpeg -i final_video.mp4 -c:v libx264 -crf 23 -preset medium -c:a aac -b:a 128k final_video_compressed.mp4
```

- **YouTube**: 1080p, H.264, 8-12 Mbps
