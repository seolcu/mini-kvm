# Mini-KVM 소프트콘 영상 내레이션 대본

**[슬라이드 1-2: 타이틀 페이지]**

안녕하십니까. 소프트웨어학과 김상훈 교수님의 지도를 받아 자기주도프로젝트로 진행한 "Mini-KVM: 교육용 하이퍼바이저"를 주제로 발표를 진행할 소프트웨어학과 설규원입니다.

본 프로젝트는 Linux KVM API를 활용하여 x86 아키텍처 상에서 완전하게 동작하는 하이퍼바이저를 구현한 연구입니다.

## 0:30-1:40 | 문제 정의 + 해결책
**[슬라이드 2-5: 가상화 기술은? / 전통적 하이퍼바이저 / Mini-KVM의 접근 / 아키텍처]**

가상화 기술은 정말 복잡해야만 할까요? 
====
QEMU는 대표적인 하이퍼바이저로, 10만 줄이 넘는 코드로 구성되어 있습니다. 초기화에 50밀리초, 메모리는 50메가바이트 이상을 사용합니다. 그런데 교육이나 임베디드 환경에서는 이러한 복잡성이 장벽이 됩니다. 코드를 이해하기 어렵고, 리소스도 과도하게 사용하죠.
====
이 문제를 해결하기 위해 Mini-KVM은 세 가지 핵심 원칙을 따릅니다. 첫째, 최소화된 복잡성으로 꼭 필요한 기능만 구현했습니다. 둘째, Linux KVM API를 활용해 하드웨어 가상화 기능을 그대로 사용합니다. 셋째, 읽기 쉬운 1,500줄의 명확한 코드로 작성했습니다.

그 결과, 초기화는 10배 빠르고, 실행 속도는 50배에서 100배 빠릅니다. 메모리는 30분의 1만 사용합니다.
====
화면의 아키텍처를 보시면 구조가 명확합니다. 맨 아래 Host Linux Kernel의 KVM 모듈이 하드웨어 가상화를 담당하고, 그 위에 Mini-KVM VMM이 vCPU 관리, 메모리 관리, VM Exit 처리를 수행합니다. 가장 위에는 최대 4개의 게스트 프로그램이 실행됩니다.

## 1:40-2:15 | 핵심 설계 혁신
**[슬라이드 6: 핵심 설계 혁신]**

Mini-KVM의 핵심적인 기술적 혁신 사항은 다음 세 가지로 요약됩니다. 첫째, 조건부 IRQCHIP 생성 메커니즘입니다. Real Mode 환경에서는 인터럽트 컨트롤러를 생성하지 않아 불필요한 오버헤드를 제거하고, Protected Mode에서만 선택적으로 IRQCHIP을 활성화하여 인터럽트 처리를 지원합니다. 둘째, vCPU별 메모리 격리 기법입니다. 각 가상 CPU는 독립적인 물리 메모리 영역을 할당받아 실행 환경의 격리성을 보장합니다. 셋째, 효율적인 Hypercall 인터페이스 구현입니다. Port 0x500을 통한 I/O 기반 통신 방식을 채택하여 게스트와 호스트 간의 경량화된 통신을 가능하게 하였습니다.

## 2:15-2:50 | Real Mode 데모 (사진)
**[슬라이드 7: Real Mode 데모 - 멀티 vCPU 병렬 실행]**

Real Mode 환경에서는 16비트 게스트 프로그램을 실행할 수 있습니다. 가장 최소화된 게스트 바이너리의 경우 1바이트 크기로 구성되며, 실행 완료까지 10밀리초 이내의 시간만을 요구합니다.

현재 화면의 스크린샷에서 볼 수 있듯이, 시스템의 핵심 강점은 멀티 vCPU 환경에서의 병렬 실행 능력입니다. 곱셈표를 계산하는 가상 머신과 숫자 카운팅을 수행하는 가상 머신이 동시에 실행되고 있으며, 출력이 문자 단위로 인터리빙되는 것을 통해 진정한 의미의 병렬 처리가 이루어지고 있음을 확인할 수 있습니다. 4개의 vCPU를 동시에 실행하는 환경에서도 90% 이상의 확장성을 유지합니다.

## 2:50-3:30 | Protected Mode 데모 (사진)
**[슬라이드 8: Protected Mode 1K OS 데모]**

Protected Mode 환경에서는 완전한 기능을 갖춘 운영체제의 실행이 가능합니다. 현재 화면에 보이는 1K OS는 RISC-V 아키텍처에서 x86으로 포팅된 교육용 운영체제로서, 9개의 사용자 프로그램을 내장하고 있습니다.

시스템 초기화 과정에서 Global Descriptor Table과 Interrupt Descriptor Table을 구성하였으며, 4메가바이트 PSE 페이징을 활성화하여 메모리 관리를 수행합니다. 타이머 인터럽트와 키보드 인터럽트 처리 메커니즘이 정상적으로 동작하며, 곱셈표 출력, 피보나치 수열 계산, 에코 프로그램 등 다양한 사용자 애플리케이션을 실행할 수 있습니다. 각 사용자 프로그램은 커널과 명확히 분리된 사용자 모드에서 실행되어 시스템의 안정성과 보안성을 확보하였습니다.

## 3:30-4:30 | 성능 비교
**[슬라이드 9-10: 성능 비교 레이더 차트 / 성능 수치 요약]**

화면의 레이더 차트는 Mini-KVM의 성능적 우위를 정량적으로 보여줍니다. 초기화 시간은 기존 대비 10배 개선되었으며, 실행 속도는 50배에서 100배에 이르는 향상을 보입니다. 메모리 사용량은 30분의 1 수준으로 감소하였고, 코드 크기는 70분의 1로 축소되었습니다. 멀티 vCPU 환경에서의 확장성은 90% 이상을 달성하였으며, 교육적 활용 가치 측면에서도 탁월한 장점을 보유하고 있습니다.

화면에 제시된 코드는 본 프로젝트의 핵심 혁신인 조건부 IRQCHIP이 어떻게 구현되는지를 보여줍니다. Real Mode 게스트는 인터럽트 처리가 불필요하므로 IRQCHIP을 생성하지 않으며, Protected Mode에서만 선택적으로 IRQCHIP을 활성화함으로써 불필요한 시스템 오버헤드를 제거하였습니다. 이러한 모든 기능과 최적화가 총 1,500줄의 소스 코드로 구현되었다는 점에서 본 프로젝트의 효율성을 확인할 수 있습니다.

## 4:30-5:00 | 응용 분야 + 마무리
**[슬라이드 11-14: 응용 분야 / 기술 스택 / 프로젝트 통계 / 가치 제안 / 마무리]**

Mini-KVM은 다양한 분야에서 실용적의 활용 가능성을 보유하고 있습니다. 교육 환경에서는 간결하고 이해하기 쉬운 코드 구조를 통해 가상화 기술의 학습 도구로 활용될 수 있으며, 리소스가 제한된 임베디드 시스템 환경에서는 최소한의 자원으로 가상 머신을 실행하는 플랫폼으로 사용 가능합니다. 또한 하이퍼바이저 연구 분야에서는 실험적 플랫폼으로 활용될 수 있으며, 개발 환경에서는 빠른 가상 머신 프로비저닝이 필요한 상황에 적용할 수 있습니다.

본 프로젝트는 복잡성을 최소화하면서도 하이퍼바이저의 핵심 기능을 완전하게 구현하였으므로, 교육용 목적으로 설계하였음에도 불구하고 실용적인 성능 수준을 달성하였습니다. MIT 라이선스를 통해 오픈소스로 공개되어 누구나 자유롭게 사용, 수정, 배포할 수 있습니다.

Mini-KVM 프로젝트는 가상화 기술의 핵심 원리를 1,500줄의 코드로 구현하여 그 본질을 증명하였습니다. 프로젝트의 전체 소스 코드와 문서는 GitHub 저장소에서 확인하실 수 있습니다.
