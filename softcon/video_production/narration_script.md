## 0:30-1:40 | 문제 정의 + 해결책 + 아키텍처
**[슬라이드 2: 가상화 기술은 복잡해야 할까요?]**

```
가상화 기술은 반드시 복잡한 구조를 가져야만 하는 것일까요?

[슬라이드 3: 전통적 하이퍼바이저의 복잡성]

대표적인 하이퍼바이저인 QEMU의 경우 10만 줄 이상의 소스 코드로 구성되어 있으며, 초기화 시간으로 약 50밀리초, 메모리 사용량으로 50메가바이트 이상을 요구합니다. 이러한 복잡성은 교육 환경이나 임베디드 시스템과 같이 리소스가 제한된 환경에서 상당한 제약 요인으로 작용하게 됩니다.

[슬라이드 4: Mini-KVM의 접근]

본 프로젝트에서는 이러한 문제를 해결하기 위해 세 가지 핵심 설계 원칙을 적용하였습니다. 첫째, 복잡성을 최소화하여 하이퍼바이저의 필수 기능만을 선별적으로 구현하였습니다. 둘째, Linux KVM API를 활용함으로써 하드웨어 가상화 기능을 직접 활용하였습니다. 셋째, 전체 코드를 1,500줄 이내로 작성하여 가독성과 유지보수성을 확보하였습니다.

이러한 설계 방식의 결과로, 초기화 시간은 기존 대비 10배 향상되었으며, 실행 속도는 50배에서 100배까지 개선되었습니다. 메모리 사용량은 30분의 1 수준으로 감소하였습니다.

[슬라이드 5: 아키텍처 개요]

화면의 아키텍처 다이어그램에서 확인할 수 있듯이, 시스템은 계층적 구조로 설계되었습니다. 최하단의 Host Linux Kernel에 포함된 KVM 모듈이 하드웨어 가상화를 담당하며, 그 상위에 위치한 Mini-KVM VMM이 vCPU 관리, 메모리 관리, VM Exit 처리 등의 핵심 기능을 수행합니다. 최상위 계층에서는 최대 4개의 게스트 프로그램이 동시에 실행될 수 있습니다.

## 1:40-2:40 | 핵심 설계 혁신 + Real Mode 데모
**[슬라이드 6: 핵심 설계 혁신]**

```
Mini-KVM의 핵심적인 기술적 혁신 사항은 다음 세 가지로 요약됩니다.

첫째, 조건부 IRQCHIP 생성 메커니즘입니다. Real Mode 환경에서는 인터럽트 컨트롤러를 생성하지 않아 불필요한 오버헤드를 제거하고, Protected Mode에서만 선택적으로 IRQCHIP을 활성화하여 인터럽트 처리를 지원합니다.

둘째, vCPU별 메모리 격리 기법입니다. 각 가상 CPU는 독립적인 물리 메모리 영역을 할당받아 실행 환경의 격리성을 보장합니다.

셋째, 효율적인 Hypercall 인터페이스 구현입니다. Port 0x500을 통한 I/O 기반 통신 방식을 채택하여 게스트와 호스트 간의 경량화된 통신을 가능하게 하였습니다.

[데모 시작 - Real Mode 단일 vCPU 5초]

Real Mode 환경에서는 16비트 게스트 프로그램을 실행할 수 있습니다. 가장 최소화된 게스트 바이너리의 경우 1바이트 크기로 구성되며, 실행 완료까지 10밀리초 이내의 시간만을 요구합니다.

[데모 시작 - Real Mode 멀티 vCPU 10초]

시스템의 핵심 강점은 멀티 vCPU 환경에서의 병렬 실행 능력입니다. 현재 화면에서는 곱셈표를 계산하는 가상 머신과 숫자 카운팅을 수행하는 가상 머신이 동시에 실행되고 있으며, 출력이 문자 단위로 인터리빙되는 것을 통해 진정한 의미의 병렬 처리가 이루어지고 있음을 확인할 수 있습니다. 4개의 vCPU를 동시에 실행하는 환경에서도 90% 이상의 확장성을 유지합니다.
```

## 2:40-3:50 | Protected Mode - 1K OS 데모
**[슬라이드 계속 또는 데모 화면]**

```
[데모 시작 - Protected Mode 1K OS 실행]

Protected Mode 환경에서는 완전한 기능을 갖춘 운영체제의 실행이 가능합니다. 현재 실행 중인 1K OS는 RISC-V 아키텍처에서 x86으로 포팅된 교육용 운영체제로서, 9개의 사용자 프로그램을 내장하고 있습니다.

시스템 초기화 과정에서 Global Descriptor Table과 Interrupt Descriptor Table을 구성하였으며, 4메가바이트 PSE 페이징을 활성화하여 메모리 관리를 수행합니다. 타이머 인터럽트와 키보드 인터럽트 처리 메커니즘이 정상적으로 동작하며, 곱셈표 출력, 피보나치 수열 계산, 에코 프로그램 등 다양한 사용자 애플리케이션을 실행할 수 있습니다. 각 사용자 프로그램은 커널과 명확히 분리된 사용자 모드에서 실행되어 시스템의 안정성과 보안성을 확보하였습니다.

[데모 진행 중 - 20초간 프로그램 실행, 내레이션 없이 타이핑/실행 소리만]
```

## 3:50-4:50 | 성능 비교 + 코드 스니펫
**[슬라이드 7: 성능 비교 레이더 차트]**

```
화면의 레이더 차트는 Mini-KVM의 성능적 우위를 정량적으로 보여줍니다. 초기화 시간은 기존 대비 10배 개선되었으며, 실행 속도는 50배에서 100배에 이르는 향상을 보입니다. 메모리 사용량은 30분의 1 수준으로 감소하였고, 코드 크기는 70분의 1로 축소되었습니다. 멀티 vCPU 환경에서의 확장성은 90% 이상을 달성하였으며, 교육적 활용 가치 측면에서도 탁월한 장점을 보유하고 있습니다.

[슬라이드 8: 코드 스니펫 - Conditional IRQCHIP]

화면에 제시된 코드는 본 프로젝트의 핵심 기술인 조건부 IRQCHIP 생성 메커니즘의 구현 방식을 보여줍니다. Real Mode 게스트는 인터럽트 처리가 불필요하므로 IRQCHIP을 생성하지 않으며, Protected Mode에서만 선택적으로 IRQCHIP을 활성화함으로써 불필요한 시스템 오버헤드를 효과적으로 제거하였습니다.

[슬라이드 9: 성능 수치 요약 테이블]

이러한 모든 기능과 최적화가 총 1,500줄의 소스 코드로 구현되었다는 점에서 본 프로젝트의 효율성을 확인할 수 있습니다.
